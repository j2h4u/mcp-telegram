## Critical Gaps

### TOOL-06: SearchMessages missing ±3 context window

**Requirement:** SearchMessages → results + ±3 context messages
**Evidence:** `search_messages` in `tools.py:338-349` uses `iter_messages(search=...)` returning only matched messages. No second pass to fetch surrounding messages.
**Impact:** LLM sees isolated matched lines without conversational context — primary value of the feature is absent.
**Phase 2 SUMMARY claimed:** "SearchMessages rewritten with name resolution, ±3 context window per hit" — this was incorrect. Phase 2 VERIFICATION marked it passed without verifying context fetch in code.
**Fix required:** Fetch `±3` messages around each hit's `id` using `client.iter_messages(entity_id, limit=7, min_id=hit.id-4, max_id=hit.id+3)` or similar.
