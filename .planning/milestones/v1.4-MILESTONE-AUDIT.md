### E2E Flows (5/5 complete)

1. **ListMessages pagination + cache + prefetch**: page 1 → cache population → page 2 cache hit → prefetch next + oldest → delta refresh on cache hit
2. **SearchMessages → cache population**: search always hits API → results written to cache → subsequent ListMessages cache hit
3. **Edit detection flow**: live fetch with changed text → `_record_versions_if_changed` → old text saved → `[edited HH:mm]` shown on next read
4. **ListUnreadMessages bypass**: zero cache code — always fetches live (BYP-03 by structural absence)
5. **ListDialogs metadata**: members/created fields conditionally emitted for group/channel entities

### Notable Observation (not a bug)

PRE-01 and PRE-03 share the `is_first_page=True` branch in `_schedule_prefetch_tasks`, differentiated only by `cache_direction`. The `is_first_page` condition conflates "newest" and "oldest" — correct behavior, but worth noting for future maintenance.
