    Add two functions to analytics.py (after TelemetryCollector class):

    1. `async def cleanup_analytics_db(db_path: Path, retention_days: int = 30) -> None:`
       Async wrapper that delegates to _sync_cleanup() on thread pool (non-blocking).
       ```python
       loop = asyncio.get_event_loop()
       await loop.run_in_executor(None, _sync_cleanup, db_path, retention_days)
       ```

    2. `def _sync_cleanup(db_path: Path, retention_days: int) -> None:`
       Synchronous cleanup (runs on thread pool):
       - Calculate cutoff_timestamp = time.time() - (retention_days * 86400) = 30 days in seconds
       - Open connection to analytics.db at db_path
       - DELETE FROM telemetry_events WHERE timestamp < ?
       - Execute PRAGMA optimize (rebuild statistics for query planner)
       - Execute PRAGMA incremental_vacuum(1000) to free up to 1000 pages without blocking
       - Commit and close connection
       - Log result: "Analytics cleanup: deleted {deleted} events, optimized, vacuumed"

    Use the pattern from research document (Section "Code Examples: Daily Cleanup Maintenance Function"). Set default retention_days=30 per REQUIREMENTS.md CACHE-03.

    **Why:** Async execution ensures cleanup doesn't block tool handlers. incremental_vacuum is non-blocking (full VACUUM would lock entire DB). PRAGMA optimize after DELETE rebuilds statistics so indexes remain effective.
