  <action>
Create analytics.py module with:

1. **TelemetryEvent dataclass** (immutable, frozen=True):
   - tool_name: str (categorical: ListDialogs, ListMessages, SearchMessages, GetMe, GetUserInfo)
   - timestamp: float (UNIX epoch, seconds)
   - duration_ms: float (milliseconds, high precision)
   - result_count: int (items returned, 0+ safe)
   - has_cursor: bool (pagination used?)
   - page_depth: int (pages fetched in session, 1+)
   - has_filter: bool (any filter applied?)
   - error_type: Optional[str] (categorical: InvalidCursor, NotFound, Ambiguous, ConnectionError, etc. — NEVER exception message or entity IDs)

2. **TelemetryCollector singleton class**:
   - __init__(db_path: Path) → initializes DB with schema, sets up batch queue, background task tracking
   - record_event(event: TelemetryEvent) → appends to self._batch under lock, returns immediately, triggers async flush if batch >= 100
   - _init_db() → creates telemetry_events table with index on (tool_name, timestamp), enables WAL mode
   - _flush_async() → spawns background task without awaiting, stores strong reference in self._background_task
   - _async_flush(batch) → offloads to thread pool via loop.run_in_executor()
   - _write_batch(batch) → synchronous DB write (runs on thread pool)
   - get_instance(db_path) → class method, thread-safe singleton via class lock

3. **Privacy-safe schema DDL**:
   ```sql
   CREATE TABLE IF NOT EXISTS telemetry_events (
       id INTEGER PRIMARY KEY AUTOINCREMENT,
       tool_name TEXT NOT NULL,
       timestamp INTEGER NOT NULL,
       duration_ms REAL NOT NULL,
       result_count INTEGER NOT NULL,
       has_cursor BOOLEAN NOT NULL,
       page_depth INTEGER NOT NULL,
       has_filter BOOLEAN NOT NULL,
       error_type TEXT
   );
   CREATE INDEX IF NOT EXISTS idx_telemetry_tool_timestamp
   ON telemetry_events(tool_name, timestamp);
   ```

4. **Error handling**:
   - record_event() never raises (telemetry is fire-and-forget)
   - _flush_async() handles case where event loop not yet running (fallback to sync write)
   - DB write errors logged but not raised

**Reference:** Phase 6 RESEARCH.md patterns (Pattern 1: TelemetryCollector, Pattern 2: Event Schema with Zero PII).
