2. **Instrumentation pattern** (apply to each of 5 tool handlers):
   Each tool handler needs:
   - Start: `t0 = time.monotonic()` at very beginning
   - During: Compute result_count, has_cursor, page_depth, has_filter from execution flow
   - Error: Set error_type if exception caught
   - Finally: Record event in finally block (ensures recording even on error)

   ```python
   @tool_runner.register
   async def list_dialogs(args: ListDialogs) -> t.Sequence[...]:
       """List dialogs with telemetry hook."""
       t0 = time.monotonic()
       error_type = None
       result_count = 0

       try:
           logger.info("method[ListDialogs] args[%s]", args)
           # ... existing ListDialogs logic ...
           result_count = len(lines)
           result = [TextContent(type="text", text="\n".join(lines))]
       except Exception as exc:
           error_type = type(exc).__name__
           raise
       finally:
           duration_ms = (time.monotonic() - t0) * 1000
           collector = _get_analytics_collector()
           from mcp_telegram.analytics import TelemetryEvent
           collector.record_event(TelemetryEvent(
               tool_name="ListDialogs",
               timestamp=time.time(),
               duration_ms=duration_ms,
               result_count=result_count,
               has_cursor=False,
               page_depth=1,
               has_filter=False,
               error_type=error_type,
           ))

       return result
   ```

3. **Tool-specific details**:
   - **ListMessages**: page_depth should track cursor iterations (increment counter each time cursor changed/fetched). If args.cursor provided, has_cursor=True.
     - has_filter: Check if args.query_sender, args.from_peer, or text search parameters used
   - **SearchMessages**: has_filter=True (search query inherently filters)
     - page_depth: Similar cursor tracking as ListMessages
   - **GetMe**: result_count always 1 (self account)
   - **GetUserInfo**: result_count always 1 (single user)
   - **GetUsageStats**: Do NOT add telemetry hook (avoid noise)

4. **Error type categorization**:
   - Use `type(exc).__name__` to get categorical error type (e.g., "NotFound", "Ambiguous", "ConnectionError")
   - NEVER include entity ID, dialog name, or exception message in error_type
   - If exception happens during telemetry recording itself, silently continue (never raise)
