From 04-RESEARCH.md — context fetch algorithm:
```python
context_ids_needed: set[int] = set()
for hit in hits:
    for offset in range(-3, 4):
        if offset != 0:
            context_ids_needed.add(hit.id + offset)
hit_ids = {h.id for h in hits}
context_ids_needed -= hit_ids  # deduplicate: hits are hits, not context

context_msgs: dict[int, object] = {}
if context_ids_needed:
    fetched = await client.get_messages(entity_id, ids=list(context_ids_needed))
    fetched_list = fetched if isinstance(fetched, list) else [fetched]
    context_msgs = {m.id: m for m in fetched_list if m is not None}

groups = []
for hit in hits:
    before = [context_msgs[hit.id - i] for i in range(3, 0, -1)
              if (hit.id - i) in context_msgs]
    after = [context_msgs[hit.id + i] for i in range(1, 4)
             if (hit.id + i) in context_msgs]
    groups.append((hit, before, after))
```

Hit-group output format (from 04-RESEARCH.md recommendation):
```
--- hit 1/N ---
HH:mm Sender: context before
[HIT] HH:mm Sender: the hit text
HH:mm Sender: context after

--- hit 2/N ---
...
```
Apply `[HIT] ` prefix to the hit line in the formatted group text. The hit is always the
middle element; after format_messages, find the line matching the hit's time+sender prefix
and prepend "[HIT] " to it. Or simpler: wrap each group separately, then string-replace
the hit line. The research recommends: format the group (sort by id desc), then prefix the
hit line using the hit's formatted time prefix as a locator.

Simplest reliable approach — format each group separately, identify the hit line by
`f"{hit_dt.strftime('%H:%M')} {sender_name}"` prefix (same pattern formatter produces),
and replace the first matching line with `[HIT] <line>`.
