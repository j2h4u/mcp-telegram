# Phase 999.1 Plan 03: Daemon API Surface — get_my_recent_activity + fragment branch Summary

Wired two new daemon-side features: `get_my_recent_activity` API method reads `activity_comments` with scan_status tri-state from `activity_sync_state`; `_list_messages` fragment branch performs targeted `get_messages` fetch for absent/not_synced/fragment dialogs and annotates responses with `coverage='fragment'`. Client convenience method added; 6 new unit tests pass.

## What Was Built

### `src/mcp_telegram/daemon_api.py`

**New dispatch branch:** `method == "get_my_recent_activity"` routes to `_get_my_recent_activity`.

**`_get_my_recent_activity` handler:**
- Reads `activity_comments` with `LEFT JOIN entities` for `dialog_name`
- Returns per-comment blocks: `dialog_id`, `message_id`, `sent_at`, `text`, `reactions`, `reply_count`, `dialog_name`
- `dialog_name` falls back to `str(dialog_id)` when no entities row exists
- `scan_status`: `"never_run"` if `last_sync_at IS NULL`, `"in_progress"` if `backfill_complete != '1'`, `"complete"` otherwise
- `since_hours` clamped `1..8760`, `limit` clamped `1..2000` (defense-in-depth, T-999.1-07/08)

**Fragment branch in `_list_messages`:**
- Inserted BEFORE the existing not_synced reject guard
- Triggers when `current_status in (None, "not_synced", "fragment")`
- Calls `_fetch_fragment_context(dialog_id, anchor_message_id)` then delegates to `_list_messages_context_window`
- Annotates result with `coverage='fragment'`
- `access_lost` status falls through to the existing not_synced rejection

**`_fetch_fragment_context` helper:**
- `INSERT OR IGNORE INTO synced_dialogs (dialog_id, status) VALUES (?, 'fragment')` — never overwrites `synced`
- Fetches anchor + 5 messages after via `client.get_messages(entity, ids=[...])`
- Upserts via `INSERT_MESSAGE_SQL` with `asdict(extracted.message)` binding pattern (canonical sync_worker pattern)
- Network errors caught and logged as WARNING; returns silently (T-999.1-11)

### `src/mcp_telegram/daemon_client.py`

Added `get_my_recent_activity(*, since_hours=168, limit=500)` convenience method on `DaemonConnection`.

### `tests/test_daemon_api.py`

Added `_make_db_with_activity()` helper: `_make_db()` + `activity_comments` + `activity_sync_state` DDL + seeded state rows + `entities` table (needed for LEFT JOIN test coverage).

Six new tests (all use `make_server(_make_db_with_activity())`):

| Test | Covers |
|------|--------|
| `test_get_my_recent_activity_never_run` | Empty state → `scan_status='never_run'`, empty comments |
| `test_get_my_recent_activity_in_progress` | `backfill_complete='0'` + `last_sync_at` set → `in_progress` |
| `test_get_my_recent_activity_filters_by_since_hours` | `since_hours=1` excludes old rows; `complete` status |
| `test_get_my_recent_activity_joins_dialog_name` | entities LEFT JOIN populates `dialog_name` |
| `test_get_my_recent_activity_falls_back_to_str_dialog_id` | No entities row → `dialog_name == str(dialog_id)` |
| `test_get_my_recent_activity_clamps_since_hours` | `since_hours=999_999` → no crash, valid response |
