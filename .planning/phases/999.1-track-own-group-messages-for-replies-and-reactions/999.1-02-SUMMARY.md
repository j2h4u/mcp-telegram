# Phase 999.1 Plan 02: activity_sync.py — backfill + incremental + daemon wiring Summary

New `activity_sync.py` module with `run_activity_sync_loop()` daemon background task that paginates the user's own messages via `SearchRequest(InputPeerEmpty, from_id=InputPeerSelf)` and populates `activity_comments`, registered in `daemon.py` alongside `run_access_probe_loop`, covered by 7 unit tests.

## What Was Built

### `src/mcp_telegram/activity_sync.py`

Background task module with three async functions:

- **`run_activity_sync_loop(client, conn, shutdown_event, *, interval=3600.0)`** — main loop; one pass = backfill (if incomplete) + incremental (if complete); sleeps `interval` seconds between passes, interruptible via `shutdown_event`
- **`_run_backfill(client, conn, shutdown_event)`** — paginates via `SearchRequest` with `offset_id` checkpoint; upserts rows into `activity_comments`; upserts entities from `result.users + result.chats`; on empty batch sets `backfill_complete='1'` and `last_sync_at`
- **`_run_incremental(client, conn, shutdown_event)`** — reads `MAX(sent_at)` from `activity_comments`; no-ops if empty (W5 guard); iterates `client.iter_messages(entity=None, from_user="me")` newest-first; breaks on `sent_at <= max_sent_at`

Supporting helpers: `_extract_dialog_id`, `_message_to_row`, `_classify_entity`, `_upsert_entities_from_search`, `_load_state`, `_set_state`, `_normalize`

Module-level SQL constants: `INSERT_ACTIVITY_SQL`, `UPSERT_ENTITY_SQL`

### `src/mcp_telegram/daemon.py`

Added import and `_create_tracked_task` call after `access_probe_loop`:

```python
from .activity_sync import run_activity_sync_loop
# ...
_create_tracked_task(
    run_activity_sync_loop(client, conn, shutdown_event),
    name="activity_sync_loop",
)
```

Existing `for task in background_tasks: task.cancel()` shutdown block handles cleanup automatically.

### `tests/test_activity_sync.py`

Seven async tests covering all specified behaviors:

| Test | Covers |
|------|--------|
| `test_backfill_inserts_rows` | Batch upsert + `backfill_complete='1'` + `last_sync_at` set |
| `test_backfill_respects_shutdown` | Pre-set shutdown → 0 API calls, state unchanged |
| `test_backfill_floodwait_recovers` | FloodWait(0) → retry → empty batch → complete |

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] FakePeer dataclass incompatible with telethon.utils.get_peer_id()**
- **Found during:** Task 3 first test run (`test_backfill_inserts_rows`)
- **Issue:** `_extract_dialog_id` calls `telethon.utils.get_peer_id(peer)` which requires actual Telethon `Peer` types (e.g. `PeerUser`, `PeerChat`, `PeerChannel`). A plain Python dataclass `FakePeer` raises `TypeError: Cannot cast FakePeer to any kind of Peer`
- **Fix:** Replaced `FakePeer` dataclass in test helper with `from telethon.tl.types import PeerUser`; updated `_msg()` to pass `PeerUser(user_id=user_id)` as `peer_id`. This is a helper fix (not a test logic change) as the plan directed
- **Files modified:** `tests/test_activity_sync.py`
- **Commit:** c964487
