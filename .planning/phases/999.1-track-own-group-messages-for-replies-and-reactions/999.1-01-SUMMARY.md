# Phase 999.1 Plan 01: Schema v14 Migration Summary

sync.db migrated from v13 to v14 with two new tables (`activity_comments` and `activity_sync_state`) for Phase 999.1 own-message tracking, covered by four dedicated migration tests.

## What Was Built

### `activity_comments` table
Own-message archive. Stores messages sent by the account owner across all chats (populated by the upcoming ActivitySyncWorker in Plan 02). Independent of the `messages` table — not FTS-indexed.

| Column | Type | Notes |
|--------|------|-------|
| dialog_id | INTEGER NOT NULL | Part of PK |
| message_id | INTEGER NOT NULL | Part of PK |
| sent_at | INTEGER NOT NULL | Unix timestamp; indexed for since_hours queries |
| text | TEXT | Message body (nullable) |
| reactions | TEXT | JSON blob `{"emoji": count}` (nullable) |
| reply_count | INTEGER NOT NULL DEFAULT 0 | Thread reply count |
| last_synced_at | INTEGER | When this row was last refreshed (nullable) |

Index: `idx_activity_comments_sent_at ON activity_comments(sent_at DESC)` — for `GetMyRecentActivity` `since_hours` range scan.

### `activity_sync_state` table
Key/value backfill progress tracking. Seeded with three rows:

| key | initial value | Purpose |
|-----|---------------|---------|
| backfill_complete | '0' | '1' when full history scan is done |
| backfill_offset_id | '0' | Telegram message_id pagination anchor (exclusive upper bound) |
| last_sync_at | NULL | Unix timestamp of last sync run; NULL = never run |

### `synced_dialogs.status` vocabulary documentation
Added comment in sync_db.py documenting all accepted values including the new `'fragment'` status value introduced by Phase 999.1 (D-04).
