# Phase 999.1.1 Plan 01: Schema v15 — Unify Messages Table Summary

Schema v15 migration that merges activity_comments into messages (out=1), drops the activity_comments and message_cache zombie tables, and adds 'own_only' to the synced_dialogs status vocabulary.

## What Was Done

**Task 1 — sync_db.py:**
- `_CURRENT_SCHEMA_VERSION` = 15
- Removed four dead DDL constants: `_MESSAGE_CACHE_TABLE_DDL`, `_MESSAGE_CACHE_INDEX_DDL`, `_ACTIVITY_COMMENTS_DDL`, `_ACTIVITY_COMMENTS_INDEX_DDL`
- Updated v4 migration block: inlined message_cache DDL (v7 DROP still works)
- Updated v14 migration block: inlined activity_comments DDL (historical arc preserved)
- Added v15 `_migrate` block with four SQL statements:
  1. `INSERT OR IGNORE INTO messages SELECT ... FROM activity_comments` (own-only data migration)
  2. `INSERT OR IGNORE INTO synced_dialogs ... 'own_only'` (orphan dialog enrollment)
  3. `DROP TABLE IF EXISTS activity_comments`
  4. `DROP TABLE IF EXISTS message_cache`
- Updated status vocabulary comment to add 'own_only' (with escalation ordering)
- Added FTS COVERAGE NOTE and SPARSE COLUMNS NOTE per cross-AI review findings

**Task 2 — tests/test_sync_db.py:**
- Removed `test_migration_v14_activity_comments` (table is gone at v15)
- Added `test_schema_v15_drops_activity_comments` — table + index absent
- Added `test_schema_v15_drops_message_cache_permanently` — zombie DDL regression guard
- Added `test_migration_v15_copies_own_only_into_messages` — data migration round-trip
- Added `test_migration_v15_preserves_existing_messages` — INSERT OR IGNORE correctness
- Added `test_migration_v15_enrolls_own_only_but_preserves_higher_status` — watermark
- Fixed `test_migration_v14_idempotent`: replaced hardcoded `assert version == 14` with `_CURRENT_SCHEMA_VERSION`
