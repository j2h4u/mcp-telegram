## What Was Done

**Task 1 — daemon_api.py:**
- Replaced `FROM activity_comments ac` SQL with `FROM messages m LEFT JOIN entities e ON e.id = m.dialog_id`
- Added three-predicate filter `WHERE m.out = 1 AND m.is_service = 0 AND m.is_deleted = 0 AND m.sent_at >= ?`
- Row tuple reduced from 7 columns to 5 (dialog_id, message_id, sent_at, text, dialog_name)
- Comment dict drops `reactions` and `reply_count` keys — now has exactly 5 keys
- Updated docstring: "non-service, non-deleted" semantics documented
- scan_status block (reads activity_sync_state) left byte-for-byte unchanged

**Task 2 — tools/activity.py:**
- Removed `import json` (only user was `_format_reactions`)
- Deleted `_format_reactions` helper entirely
- `_format_block` simplified to 2-line output: `[dialog_name] YYYY-MM-DD HH:MM  text` + `nav: dialog_id=… message_id=…`
- `GetMyRecentActivity` docstring updated: "with reply counts and reactions" → removed

**Task 3 — tests/test_daemon_api.py:**
- `_make_db_with_activity` no longer creates activity_comments DDL; layers only activity_sync_state on `_make_db()`
- 4 tests rewritten to INSERT into `messages` with `out=1, is_service=0, is_deleted=0` instead of `activity_comments`
- New `test_get_my_recent_activity_filters_incoming_messages`: seeds all 4 predicate combinations, proves ONLY (out=1, is_service=0, is_deleted=0) row surfaces
- New `test_search_messages_finds_migrated_own_message`: D-5 unit proof — seeds out=1 row, calls `backfill_fts_index()`, asserts FTS MATCH finds the row (join on (dialog_id, message_id) since messages is WITHOUT ROWID)

### Auto-fixed Issues

**1. [Rule 1 - Bug] test_tool_routing.py reacted to old response schema**
- **Found during:** Task 4 full suite run
- **Issue:** `test_get_my_recent_activity_reactions_formatted` tested that `👍×2` and `❤×1` appeared in formatted output — but `reactions` field and `_format_reactions` were deleted in Task 2
- **Fix:** Renamed test to `test_get_my_recent_activity_formats_comment_block`, rewrote to verify 2-line format (dialog/text + nav), asserts `reactions:` and `replies:` substrings are absent
- **Files modified:** tests/test_tool_routing.py
- **Commit:** c289698

**2. [Rule 1 - Bug] FTS join used rowid instead of (dialog_id, message_id)**
- **Found during:** Task 3 test execution
- **Issue:** Initial implementation of `test_search_messages_finds_migrated_own_message` used `JOIN messages m ON m.rowid = fts.rowid` — but messages table is `WITHOUT ROWID`, causing `OperationalError: no such column: m.rowid`
- **Fix:** Changed join to `ON m.dialog_id = fts.dialog_id AND m.message_id = fts.message_id` (matching the pattern in _SELECT_FTS_SQL in daemon_api.py). Also applied `stem_query()` to the search term since FTS5 stores stemmed_text.
- **Files modified:** tests/test_daemon_api.py
- **Commit:** e7f779f (included in same commit)
